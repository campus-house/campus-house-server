📖 Swagger UI에서 API 테스트하기

💬 채팅 API

POST /api/chat/rooms
채팅방 생성/조회 - 이미 채팅방이 있으면 기존 방 반환, 없으면 새 방 생성
GET /api/chat/rooms/user/{userName}
사용자 채팅방 목록 조회
GET /api/chat/rooms/{roomId}
특정 채팅방 조회
GET /api/chat/rooms/users/{user1Name}/{user2Name}
두 사용자 간 채팅방 조회
POST /api/chat/rooms/{roomId}/messages
메시지 전송
GET /api/chat/rooms/{roomId}/messages
메시지 목록 조회 (페이징 지원)
POST /api/chat/rooms/{roomId}/read
메시지 읽음 처리
GET /api/chat/unread-count/{userName}
읽지 않은 메시지 수 조회
GET /api/chat/rooms/{roomId}/unread-count/{userName}
특정 채팅방 읽지 않은 메시지 수 조회

🔐 인증 방법

모든 채팅 API는 JWT 토큰 인증이 필요합니다.

Authorization 헤더에 "Bearer {JWT_TOKEN}" 형식으로 토큰을 포함해야 합니다.

토큰은 /api/auth/login API를 통해 획득할 수 있습니다.

📝 사용 예시

1. 로그인하여 토큰 획득

POST /api/auth/login 
{
  "email": "user@example.com",
  "password": "password123"
}

2. 채팅방 생성/조회

POST /api/chat/rooms?otherUserName=홍길동
Authorization: Bearer {JWT_TOKEN}

3. 메시지 전송

POST /api/chat/rooms/1/messages 
{
  "content": "안녕하세요!",
  "messageType": "TEXT"
}
